扫描 `examples/` 目录下所有示例，并更新项目根目录 `README.md` 中的「示例列表」表格和「项目结构」目录树。

请按以下步骤执行：

## 第一步：扫描示例目录

使用 `ls` 列出 `examples/` 下所有子目录（按名称排序），获取所有 `NN-example-name/` 格式的示例。

## 第二步：读取每个示例的信息

逐个读取每个示例目录下的 `README.md`，从中提取：
- **标题**：第一行 `#` 标题
- **一句话描述**：标题下方第一个非空段落（通常紧跟在标题后面）

## 第三步：更新示例列表表格

将 `README.md` 中 `## 示例列表` 部分的表格替换为最新内容。表格格式为：

```markdown
| 编号 | 示例名称 | 描述 |
|------|----------|------|
| 01 | [标题](examples/01-xxx/) | 一句话描述 |
```

其中：
- 编号从目录名 `NN-xxx` 中的 `NN` 提取
- 示例名称使用简短的中文名称（从 README 标题中提炼，去掉冗余的前缀如 "ESP32"、"03 -" 等）
- 描述使用 README 中的一句话描述，保持简洁
- 链接指向对应的 `examples/NN-xxx/` 目录

表格后面的提示行 `> 更多示例持续更新中...` 保留不动。

## 第四步：更新项目结构目录树

将 `README.md` 中 `## 项目结构` 下的目录树替换为最新内容。格式为：

```
esp32-examples/
├── README.md                   # 项目说明文档
├── LICENSE                     # GNU GPL v3 许可证
├── .gitignore                  # Git 忽略配置
└── examples/                   # 示例项目目录
    ├── 01-xxx/                 # 简短描述
    ├── 02-xxx/                 # 简短描述
    └── 03-xxx/                 # 简短描述
```

注意事项：
- 根目录下的文件保持不变（README.md、LICENSE、.gitignore）
- 每个示例目录后面的注释使用简短描述
- 最后一个示例使用 `└──`，其余使用 `├──`

## 第五步：确认变更

完成编辑后，展示更新后的「示例列表」和「项目结构」两部分内容，让用户确认是否正确。
